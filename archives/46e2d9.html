配置二级域名

发表于 2020-08-14 更新于 2021-09-02 分类于 服务端 Waline： 阅读次数： 本文字数： 694 阅读时长 ≈ 1 分钟

目的是将此博客的域名更改为一个二级域名，但是还有一些问题没有解决的办法，先记录下，不想半途而废，以后再改。

遇到的一些问题：首先，此博客是双地部署的，GitHub 和 Coding。

  • 第一个问题：一级域名是配置在个人的服务器上的，以 ip 地址通过 A 来解析的，而 GitHub 给的是一个 feiyeyuanye.github.io ，一般这需要使用 CNAME 来解析，但 A 和 CNAME 有了冲突。（解决办法之一是可以通过 显/隐性 URL 来解析，但是当同时将 GitHub 和 Coding 都通过 URL 来解析时，两个 URL 又有了冲突）
  • 第二个问题：本来想着，那就通过 ping feiyeyuanye.github.io 来获取 ip 地址，得到的却是 127.0.0.1，最后 GitHub 改回使用原域名也无法访问了，便放下了，没有配置。（但 Coding 通过这种方式配置成功了）
  • 第三个问题：虽然 Coding 成功了，但也出现了问题。首先，个人服务器已配置了单域名证书(这是一级域名的证书，并且单域名证书只可以保护一个域名)，而 Coding 默认也会自动配置证书(这是二级域名的证书)，最后访问网页时会出现证书配置错误的情况，比如访问二级域名的网页时，证书却是一级域名的证书。
  • 第四个问题：此博客出了许多的错误，比如统计。
  • 第五个问题：这是已经备案的域名，可能还需要加上备案信息。
  • 第六个问题：其它未知或是遗忘的问题。
